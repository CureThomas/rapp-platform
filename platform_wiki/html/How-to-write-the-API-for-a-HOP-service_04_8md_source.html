1 Currently, we support and maintain RAPP-API(s) for the following programming languages:
2 
3 - Python
4 - JavaScript
5 - C++
6 
7 Under the [rapp-api repository](https://github.com/rapp-project/rapp-api) you can find more information regarding implemented and tested Rapp-Platform Service calls.
8 
9 As the integration tests are written in Python and uses the [Python-Rapp-APi](https://github.com/rapp-project/rapp-api/tree/master/python) it is a good practice to start on the python-rapp-api.
10 
11 ### Python Rapp-API Overview
12 
13 The Python Rapp-API package can be found under the rapp-api/python/ directory, or [here](https://github.com/rapp-project/rapp-api/tree/master/python).
14 
15 The package includes the following components:
16 
17 - RappCloud.py: For each developed HOP Web Service this class includes a method that is used to call the relevant HOP Web Service.
18 - CloudInterface: Service controller that handles HOP Web Service calls. It is actually an implementation of an HTTP.Post service controller.
19 - Configuration files: Configuration files which includes information on:
20  - Platform authentication credentials, [auth.cfg](https://github.com/rapp-project/rapp-api/blob/master/python/RappCloud/config/auth.cfg). The default user for authentication is the **rapp** user. Platform-side authentication is handled by the HOP Server!!
21  - Platform address information, [platform.cfg](https://github.com/rapp-project/rapp-api/blob/master/python/RappCloud/config/platform.cfg). Here both the Platform ipv4-address and HOP Server listening port are declared.
22  - Platform HOP Web Services information, [services.yaml](https://github.com/rapp-project/rapp-api/blob/master/python/RappCloud/config/services.yaml). Each developed HOP Web Service has its own entry into this file.
23 
24 To extend the Python-Rapp-API with a new HOP Web Service call entry you have to:
25 
26 - Implement the specific RappCloud class member function.
27 - Append the relevant HOP Web Service name into the **services.yaml** file.
28 
29 
30 ### Implement a RappCloud class member function as a HOP Web Service API call.
31 
32 Lets say we want to implement the face_detection API call. The face_detection HOP Web Service has one input parameter that is a file (image frame).
33 
34 We implement a member function, named face_detection with one input parameter, named **file_uri**. This parameter is the system path to an image file we want to perform face_detection on.
35 
36 ```python
37 class RappCloud:
38  ...
39  ...
40 
41  def face_detection(self, fileUri):
42  pass
43 ```
44 
45 As you can see the above code just declares the API method and does nothing.
46 
47 HTTP .post requests can have the following fields:
48 
49  - payload: The payload declares simple {parameter, value} pairs (param1: value).
50  - files: Used to transfer files over the Web, using multipart/form-data post requests.
51 
52 So lets declare two variables into our code to describe the aforementioned fields.
53 
54 ```python
55  def face_detection(self, fileUri):
56  payload = {}
57  files = {}
58 
59 ```
60 
61 The face_detection HOP Web Service has one input parameter, named file_uri. We have to declare the **file_uri** field and append data into that field. Though keep in mind that the **file_uri** field holds the name of the trasfered, to the Platform HOP Web Service, file. To ensure that HOP Service incoming requests will never have conflicts on file_names because if this happens files on Platform will overwrite each other, we append a random identity as the file's name postfix ( **image1.png --> image1-akji12.png** ).
62 The RappCloud class holds an instance of the [Random String Generator](https://github.com/rapp-project/rapp-api/blob/master/python/RappCloud/RandStrGen/RandStrGen.py) class to perform these kind of operations.
63 To append a unique postfix to the file's name call the **__appendRandStr** member:
64 
65 ```python
66  def face_detection(self, fileUri):
67  fileName = self.__appendRandStr(fileUri)
68  payload = {}
69  files = {
70  'file_uri': (fileName, open(fileUri, 'rb'))
71  }
72 
73 ```
74 
75 Notice that we leave the payload to be an empty field!!!
76 
77 Finally we need call the CloudInterface service controller. RappCloud class holds an instance of the CloudInterface class.
78 
79 ```python
80  def face_detection(self, fileUri):
81  fileName = self.__appendRandStr(fileUri)
82  payload = {}
83  files = {
84  'file_uri': (fileName, open(fileUri, 'rb'))
85  }
86 
87  url = self.serviceUrl_['face_detection']
88 
89  returnData = CloudInterface.callService(url, payload, files, self.auth_)
90  return returnData
91 
92 ```
93 
94 Though one last step is required in order to make the above code execute correctly
95 We need to add the Face-Detection HOP Web Service name into the [services.yaml](https://github.com/rapp-project/rapp-api/blob/master/python/RappCloud/config/services.yaml) file.
96 
97 ```yaml
98 
99 services:
100  list:
101  - face_detection
102 
103  ...
104 ```
105 
106 Make sure to check the Platform Authentication credentials ([auth.cfg](https://github.com/rapp-project/rapp-api/blob/master/python/RappCloud/config/auth.cfg)) and Platform Parameters ([platform.cfg](https://github.com/rapp-project/rapp-api/blob/master/python/RappCloud/config/platform.cfg)).
107 
108 By default, Platform Parameters point to the locally installed RAPP-Platform. Read the **platfrom.cfg** file on how to change these parameters, if for example you installed the RAPP-Platform on a remote machine.
