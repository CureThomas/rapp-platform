rapp-platform.wiki/Create-an-in-robot-application-for-NAO-in-Python-(novice).md
Go to the documentation of this file.
1 This tutorial has a goal to introduce a novice programmer (who is not necessarily a roboticist) into the RApps concept (where RApps stands for Robotic Applications). Here, a simple application will be created for the NAO robot using the Python programming language. This application will be executed in-robot, thus a real NAO robot is necessary.
2 
3 ##Preparation steps
4 
5 For this tutorial we will use the following tools:
6 - A real NAO robot
7 - the ```rapp-robots-api``` Github repository
8 
9 Of course the standard prerequisites are a functional installation of Ubuntu 14.04, an editor and a terminal.
10 
11 ####RAPP Robots API libraries setup
12 
13 The first step is to clone the appropriate GitHub repository in your PC:
14 
15 ```bash
16 mkdir ~/rapp_nao
17 cd ~/rapp_nao
18 git clone https://github.com/rapp-project/rapp-robots-api.git
19 ```
20 
21 The next step is to transfer the RAPP Python libraries to the NAO robot. This will be done via ```scp```, assuming that the NAO robot's IP is ```192.168.0.101``` and username and password are ```nao```:
22 
23 ```bash
24 cd ~/rapp_nao/
25 tar -zcvf rapp_api.tar.gz rapp-robots-api/
26 scp rapp_api.tar.gz nao@192.168.0.101:~/rapp_api.tar.gz
27 ```
28 
29 Now connect in NAO via ssh by ```ssh nao@192.168.0.101``` giving ```nao``` as password. Then untar the API:
30 
31 ```bash
32 tar -xvf rapp_api.tar.gz
33 rm rapp_api.tar.gz
34 ```
35 
36 The next step is to update the ```PYTHONPATH``` variable. Since NAO has Gentoo as OS, we will modify the ```bash_profile``` file:
37 
38 ```bash
39 echo 'export PYTHONPATH=$PYTHONPATH:~/rapp-robots-api/python/abstract_classes' >> /home/nao/.bash_profile
40 echo 'export PYTHONPATH=$PYTHONPATH:~/rapp-robots-api/python/implementations/nao_v4_naoqi2.1.4' >> /home/nao/.bash_profile
41 source ~/.bash_profile
42 ```
43 
44 The last step to configure the ```rapp-robots-api``` is to declare the NAO IP. Since the robot API is in-robot, the IP must be localhost: ```127.0.0.1```.
45 
46 The IP must be declared in the first line of [this](https://github.com/rapp-project/rapp-robots-api/blob/master/python/implementations/nao_v4_naoqi2.1.4/nao_connectivity) file, thus the ```nao_connectivity``` file located under ```/home/nao/rapp-robots-api/python/implementations/nao_v4_naoqi2.1.4/nao_connectivity``` should contain:
47 
48 ```
49 127.0.0.1
50 9559
51 ```
52 
53 Now all tools are in place to write our simple NAO Python application.
54 
55 ##Writing a simple application
56 
57 Let's create a Python file for our application and give it execution rights:
58 
59 ```bash
60 mkdir ~/rapp_nao
61 cd /home/nao/rapp_nao
62 mkdir rapps && cd rapps
63 touch simple_app.py
64 chmod +x simple_app.py
65 ```
66 
67 The first step is to check if everything is in place. Write the following in the ```simple_app.py``` file:
68 
69 ```python
70 #!/usr/bin/env python
71 from rapp_robot_api import RappRobot
72 rh = RappRobot()
73 rh.audio.speak("Hello there!")
74 ```
75 
76 If everything was set-up correctly the NAO robot should talk and say "Hello there!" to you. If not, one of the aforementioned instructions was not performed correctly (or if it they all were, please submit a bug to correct this tutorial!).
77 
78 Now for the real application, you can use any of the documented API calls that exist [here](https://github.com/rapp-project/rapp-robots-api/tree/master/python). Insert the following in the ```simple_app.py``` file:
79 
80 ```python
81 #!/usr/bin/env python
82 
83 # Import the RAPP Robot API
84 from rapp_robot_api import RappRobot
85 # Create an object in order to call the desired functions
86 rh = RappRobot()
87 
88 # Adjust the NAO master volume and ask for instructions. The valid commands are 'stand' and 'sit' and NAO waits for 5 seconds
89 rh.audio.setVolume(50)
90 rh.audio.speak("Hello there! What do you want me to do? I can sit or get up.")
91 res = rh.audio.speechDetection(['sit', 'get up'], 5)
92 print res
93 word = ''
94 inner_word = ''
95 if res['error'] == None:
96  word = res['word']
97 
98 # Check which command was dictated by the human
99 if word == 'sit':
100  # The motors must be enabled for NAO to move
101  rh.motion.enableMotors()
102  # NAO sits with 75% of its maximum speed
103  rh.humanoid_motion.goToPosture('Sit', 0.75)
104 elif word == 'get up':
105  # The motors must be enabled for NAO to move
106  rh.motion.enableMotors()
107  # NAO stands with 75% of its maximum speed
108  rh.humanoid_motion.goToPosture('Stand', 0.75)
109 else:
110  # No command was dictated or the command was not understood
111  pass
112 
113 # Ask the human what movement to do: move the hands or the head?
114 rh.audio.speak("Do you want me to move my arms or my head?")
115 res = rh.audio.speechDetection(['arms', 'head'], 5)
116 print res
117 if res['error'] == None:
118  word = res['word']
119 
120 rh.motion.enableMotors()
121 if word == 'arms':
122  rh.audio.speak("Do you want me to open the left or right hand?")
123  res = rh.audio.speechDetection(['left', 'right'], 5)
124  print res
125  if res['error'] == None:
126  inner_word = res['word']
127  if inner_word == 'left':
128  rh.humanoid_motion.openHand('Left')
129  elif inner_word == 'right':
130  rh.humanoid_motion.openHand('Right')
131  else:
132  pass
133 
134  rh.audio.speak("I will close my hands now")
135  rh.humanoid_motion.closeHand('Right')
136  rh.humanoid_motion.closeHand('Left')
137 elif word == 'head':
138  rh.audio.speak("Do you want me to turn my head left or right?")
139  res = rh.audio.speechDetection(['left', 'right'], 5)
140  print res
141  if res['error'] == None:
142  inner_word = res['word']
143  # The head moves by 0.4 rads left or right with 50% of its maximum speed
144  if inner_word == 'left':
145  rh.humanoid_motion.setJointAngles(['HeadYaw'], [0.4], 0.5)
146  elif inner_word == 'right':
147  rh.humanoid_motion.setJointAngles(['HeadYaw'], [-0.4], 0.5)
148  else:
149  pass
150 
151  rh.audio.speak("I will look straight now")
152  rh.humanoid_motion.setJointAngles(['HeadYaw'], [0], 0.5)
153 else:
154  pass
155 
156 rh.audio.speak("And now I will sit down and sleep!")
157 rh.humanoid_motion.goToPosture('Sit', 0.7)
158 rh.motion.disableMotors()
159 ```
160 
161 As you may have noticed the API calls are robot-agnostic, meaning that the developer (you) can create applications without having to specify on which robot they will be executed. If a specific function is not implemented in a robot that will execute this application, the specific command will simply have no effect.
162 
163 Finally, the last step is to execute the RApp:
164 
165 ```bash
166 python /home/nao/rapp_nao/rapps/simple_app.py
167 ```
