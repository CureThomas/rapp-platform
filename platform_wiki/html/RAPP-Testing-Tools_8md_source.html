1 Testing tools used for RAPP Platform integration tests.
2 
3 The RAPP Testing Tools have been developed in consideration of providing to **RAPP developers**, a system integration suite.
4 While developing independent RIC modules, Web Services, Platform Agents, Robot Agents, the rapp_testing_tools package provides a way to test the functionality, system-wide.
5 
6 All tests use the Python implementation of the RAPP Platform API,[rapp-platform-api/python](https://github.com/rapp-project/rapp-api/tree/master/python), to request for .
7 
8 A test, written under the RAPP testing tools framework, is composed of, at-least, the following ingredients:
9 
10 - The source code of the test class.
11 - (**Optional**) Data files used as a part of the test, if any are used. Data files can be, for example, image data files that will be transmitted to the RAPP Platform in order to perform image processing algorithms on those.
12 
13 The implementation of a test is splitted into the following:
14 
15 - The RAPP Platform Service call(s).
16 - The validation of the response.
17 
18 Test classes inherit from Python unit testing framework, [unittest](https://docs.python.org/2.7/library/unittest.html)
19 
20 At least one test, for each module located under the RAPP Improvement Center, has been developed.
21 Integration test are used to check on the integration functionality, of the independent components which constitute the RAPP system. Those are developed in a way to also test the functionality of the interface layers between a client-side process and the RAPP Services.

24 ## Executing integration tests
25 
26 
27 The rapp_run_test.py script is used in order to execute developed tests.
28 This script accepts the following arguments passed by the cli:
29 
30 - [ **-i** ] : Test filepath, to execute. If empty, all tests under the **default_tests** directory will be executed.
31 
32 ```bash
33 $ ./rapp_run_test_.py -i default_tests/face_detection_tests.py
34 ```
35 
36 - [ **-n** ] : Number of times of test(s) execution.
37 
38 ```bash
39 $ ./rapp_run_test.py -i default_tests/face_detection_tests.py -n 10
40 ```
41 
42 - [ **-t** ] : Run the test in threaded mode. Multible invocations can be done, simultaneous. Each test execution is handled by a single thread.
43 
44 ```bash
45 $ ./rapp_run_test.py -i default_tests/face_detection_tests.py -n 10 -t
46 ```
47 
48 **Note:**
49 If no test_name is provided as argument, all tests are executed!!

53 ## Developing integration tests
54 
55 A template_test.py is located under the [default_tests](https://github.com/rapp-project/rapp-platform/tree/master/rapp_testing_tools/scripts/default_tests) directory of the rapp_testing_tools package.
56 
57 ```bash
58 $ <path_to_rapp_testing_tools_package>/scripts/default_tests/
59 ```
60 
61 Also, several tests have already been developed, located under the default_tests
62 directory. Those can be used as a reference/examples for developing new tests.
63 
64 
65 A Section in the [Wiki](https://github.com/rapp-project/rapp-platform/wiki/How-to-implement-an-integration-test%3F) exists that guides through developing integration tests for the RAPP Platform.
