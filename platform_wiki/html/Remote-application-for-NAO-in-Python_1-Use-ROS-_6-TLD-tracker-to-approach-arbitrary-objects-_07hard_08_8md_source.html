Go to the documentation of this file.
1 In the current tutorial we will showcase the way to create a more complex application which utilizes external (out of RAPP) packages. This application will be create as a ROS node, in order to take advantage of other ROS packages.
2 
3 In this application we will track an object using the NAO front camera and the [OpenTLD](https://github.com/pandora-auth-ros-pkg/open_tld) algorithm, as well as [its wrapper](https://github.com/pandora-auth-ros-pkg/pandora_tld) used in the [PANDORA robotics team](http://pandora.ee.auth.gr/).
4 
5 Thus for this tutorial we will need:
6 - A real NAO robot
7 - the NaoQi Python libraries
8 - the rapp-robots-api GitHub repository
9 - the OpenTLD GitHub repository
10 - the PANDORA OpenTLD wrapper
11 - ROS Indigo installed
12 - ROS packages for NaoQi
13 
14 This application's code can be found [here](https://github.com/rapp-project/rapps-nao/tree/master/3.track_object_TLD/tld_tracker_nao) in the form of a ROS package.
15 
16 ##Preparation
17 
18 ####Setup the NaoQi Python libraries

22 ###Setup the RAPP Robots Python API
23 
24 Check [here](https://github.com/rapp-project/rapp-platform/wiki/Create-a-remote-robotic-application-for-NAO-in-Python-(novice)#rapp-robots-api-libraries-setup) for details.

26 ####Install ROS Indigo and ROS packages for NaoQi
27 
28 The installation procedure can be found [here](http://wiki.ros.org/indigo/Installation/Ubuntu) for the Ubuntu 14.04 distribution.
29 
30 After ROS installation, install the NaoQi ROS packages by:
31 
32 ```bash
33 sudo apt-get install ros-indigo-naoqi-*
34 sudo apt-get install ros-indigo-nao-bringup ros-indigo-nao-apps ros-indigo-usb-cam
35 ```

37 ####Setup OpenTLD and PANDORA TLD ROS packages
38 
39 Initially, a ```catkin_repository``` must be created and the ROS packages cloned in it:
40 
41 ```bash
42 # Create the proper folders
43 mkdir -p ~/rapp_nao/rapps && cd ~/rapp_nao/rapps
44 mkdir -p rapp_catkin_ws/src && cd rapp_catkin_ws/src
45 # Initialize the Catkin workspace
46 catkin_init_workspace
47 # Clone the OpenTLD and PANDORA TLD packages
48 git clone https://github.com/pandora-auth-ros-pkg/pandora_tld.git
49 git clone https://github.com/pandora-auth-ros-pkg/open_tld.git
50 # Build the packages
51 cd ~/rapp_nao/rapps/rapp_catkin_ws && catkin_make -j1
52 # Export the proper environmental variables
53 echo 'source ~/rapp_nao/rapps/rapp_catkin_ws/devel/setup.bash --extend' >> ~/.bashrc
54 source ~/.bashrc
55 ```
56 
57 Next, you must change the input image topic to TLD in order to bind the callback from the NAO camera. The ```~/rapp_nao/rapps/rapp_catkin_ws/src/pandora_tld/config/predator_topics.yaml``` must include the following:
58 
59 ```yaml
60 predator_alert : /vision/predator_alert
61 input_image_topic: /nao_robot/camera/top/camera/image_raw
62 begin_hunt_topic: /predator/hunt
63 ```
64 ##Creating the application
65 
66 Initially a ROS package must be created that will facilitate the code. The package will have ```rospy``` as dependency and will be named ```tld_tracker_nao```:
67 
68 ```bash
69 cd ~/rapp_nao/rapps/rapp_catkin_ws/src
70 catkin_create_pkg tld_tracker_nao rospy geometry_msgs sensor_msgs std_msgs
71 mkdir -p tld_tracker_nao/scripts
72 touch tld_tracker_nao/scripts/tracker.py
73 chmod +x tld_tracker_nao/scripts/tracker.py
74 ```
75 
76 The contents of ```tracker.py``` follow:
77 
78 ```python
79 #!/usr/bin/env python
80 
81 # Authors:
82 # Chrisa Gouniotou (https://github.com/chrisagou)
83 # Aspa Karanasiou (https://github.com/aspa1)
84 # Manos Tsardoulias (https://github.com/etsardou)
85 
86 # Import the RAPP Robot API
87 from rapp_robot_api import RappRobot
88 
89 from geometry_msgs.msg import Polygon
90 from geometry_msgs.msg import Twist
91 
92 from naoqi_bridge_msgs.msg import JointAnglesWithSpeed
93 
94 import rospy
95 import sys
96 import time
97 
98 class NaoTldTracker:
99 
100  def __init__(self):
101  self.rh = RappRobot()
102 
103  # Use naoqi_brdge to move the head
104  self.joint_pub = rospy.Publisher('/joint_angles', JointAnglesWithSpeed, queue_size=1)
105 
106  # NAO stands up
107  self.rh.motion.enableMotors()
108  self.rh.humanoid_motion.goToPosture("Stand", 0.7)
109 
110  self.lost_object_counter = 20
111  self.lock_motion = False
112  self.hunt_initiated = False
113 
114  # These are the NAO body velocities. They are automatically published
115  # in the self.set_velocities_callback
116  self.x_vel = 0
117  self.y_vel = 0
118  self.theta_vel = 0
119 
120  # Subscription to the TLD tracking alerts
121  self.predator_sub = rospy.Subscriber("/vision/predator_alert", \
122  Polygon, self.track_bounding_box)
123 
124  # Timer callback to check if tracking has been lost
125  rospy.Timer(rospy.Duration(0.1), self.lost_object_callback)
126  # Callback to set the velocities periodically
127  rospy.Timer(rospy.Duration(0.1), self.set_velocities_callback)
128 
129  # Callback of the TLD tracker. Called when the object has been detected
130  def track_bounding_box(self, polygon):
131  self.hunt_initiated = True
132 
133  # Set the timeout counter to 2 seconds
134  self.lost_object_counter = 20
135 
136  # Velocities message initialization
137  joint = JointAnglesWithSpeed()
138  joint.joint_names.append("HeadYaw")
139  joint.joint_names.append("HeadPitch")
140  joint.speed = 0.1
141  joint.relative = True
142 
143  # Get center of detected object and calculate the head turns
144  target_x = polygon.points[0].x + 0.5 * polygon.points[1].x
145  target_y = polygon.points[0].y + 0.5 * polygon.points[1].y
146 
147  sub_x = target_x - 320 / 2.0
148  sub_y = target_y - 240 / 2.0
149 
150  var_x = (sub_x / 160.0)
151  var_y = (sub_y / 120.0)
152 
153  joint.joint_angles.append(-var_x * 0.05)
154  joint.joint_angles.append(var_y * 0.05)
155 
156  ans = self.rh.humanoid_motion.getJointAngles(['HeadYaw', 'HeadPitch'])
157  head_pitch = ans['angles'][1]
158  head_yaw = ans['angles'][0]
159 
160  # Get the sonar measurements
161  sonars = self.rh.sensors.getSonarsMeasurements()
162 
163  # Check if NAO is close to an obstacle
164  if sonars['sonars']['front_left'] <= 0.3 or sonars['sonars']['front_right'] <= 0.3:
165  self.lock_motion = True
166  rospy.loginfo("Locked due to sonars")
167  # Check if NAOs head looks way too down or up
168  elif head_pitch >= 0.4 or head_pitch <= -0.4:
169  self.lock_motion = True
170  rospy.loginfo("Locked due to head pitch")
171  # Else approach the object
172  elif self.lock_motion is False:
173  self.theta_vel = head_yaw * 0.1
174  if -0.2 < head_yaw < 0.2:
175  print "Approaching"
176  self.x_vel = 0.5
177  self.y_vel = 0.0
178  else:
179  self.x_vel = 0.0
180  self.y_vel = 0.0
181  print "Centering"
182  self.joint_pub.publish(joint)
183 
184  # Check the battery levels
185  batt = self.rh.sensors.getBatteryLevels()
186  battery = batt['levels'][0]
187  if battery < 25:
188  self.rh.audio.setVolume(100)
189  self.rh.audio.speak("My battery is low")
190  self.predator_sub.unregister()
191  self.rh.humanoid_motion.goToPosture("Sit", 0.7)
192  self.rh.motion.disableMotors()
193  sys.exit(1)
194 
195  # Callback invoked every 0.1 seconds to check for lost of object tracking
196  def lost_object_callback(self, event):
197  # Continues only after the user has selected an object
198  if self.hunt_initiated:
199  self.lost_object_counter -= 1
200  # If 2 seconds have passed without tracking activity the robot stops
201  if self.lost_object_counter < 0:
202  self.lock_motion = True
203  self.x_vel = 0.0
204  self.y_vel = 0.0
205  self.theta_vel = 0.0
206  rospy.loginfo("Locked due to 2 seconds of non-tracking")
207 
208  self.predator_sub.unregister()
209 
210  # Callback to periodically (0.1 sec) set velocities, except from the
211  # case where the robot has locked its motion
212  def set_velocities_callback(self, event):
213  if not self.lock_motion:
214  self.rh.motion.moveByVelocity(self.x_vel, self.y_vel, \
215  self.theta_vel)
216  else:
217  self.rh.motion.moveByVelocity(0, 0, 0)
218 
219 # The main function
220 if __name__ == "__main__":
221  rospy.init_node('nao_tld_tracker', anonymous = True)
222  nao = NaoTldTracker()
223  rospy.spin()
224 ```

327 If everything is correct, a TLD window must be raised, showing live the stream from the NAO front camera. Click on this window, press ```r``` and select a rectangle with your mouse for the NAO to track. Have in mind that the object-to-track must have sufficient features for the TLD to succeed (e.g. a white wall won't work).
328 
329 Then, NAO should center the object with its head and start moving towards there. There are four reasons for NAO to stop:
330 - To loose tracking for 2 seconds
331 - To have a sonar measurement under 30 cm
332 - For the head pitch angle to be larger than 0.4 rad or smaller than -0.4 rad (the object is on the floor or very high)
333 - The battery level to be under 25%
