8 ## Hop.js made simple - Overview
9 
10 Hop.js is a multitier extension of JavaScript. It allows a single JavaScript program to describe the client-side and the server-side components of a Web application. Its runtime environment ensures a consistent execution of the application on the server and on the client.
11 
12 HopScript is based on JavaScript and aims at ensuring compatibility with this language
13 
14 - HopScript fully supports ECMAScript 5
15 - HopScript also support some ECMAScript 6 features:
16  - arrow functions.
17  - default function parameters.
18  - rest arguments.
19  - let and const bindings.
20  - symbols.
21  - template strings.
22  - promises.
23  - typed Arrays.
24 - HopScript supports most Nodejs APIs (see Section Nodejs)
25 
26 Hop programs execute in the context of a builtin web server. They define services, which are super JavaScript functions that get automatically invoked when HTTP requests are received. Functions and services are almost syntactically similar but the latter are defined using the service keyword:
27 
28 ```javascript
29 service hello() {
30  return "hello world";
31 }
32 ```
33 
34 To run this program put this code in the file hello.js and execute:
35 
36 ```shell
37 $ hop -p 8080 hello.js
38 ```
39 
40 Hop extends JavaScript with the geniune HTML. if we want to modify our service to make it return an HTML document, we can use:
41 
42 ```javascript
43 service hello() {
44  return <html><div>hello world</div></html>;
45 }
46 ```
47 
48 Hop is multitier. That is client-side codes are also implemented in Hop. The ~{ mark switches from server-side context to client-side context:
49 
50 ```javascript
51 service hello() {
52  return <html><div onclick=~{ alert( "world" ) }>hello</html>;
53 }
54 ```
55 
56 Hop client-side code and server-side can also be mixed using the ${ mark:
57 
58 ```javascript
59 service hello( { who: "foo" } ) {
60  return <html><div onclick=~{ alert( "Hi " + ${who} + "!") }>hello</html>;
61 }
62 ```

64 ## Integration with the ROS Framework
65 
66 In most cases a HOP Web Service must be able to "**Talk ROS**", meaning that a stable interface between the two, HOP Web Service and ROS, must exist.
67 
68 For the RAPP Platform requirements, we use the lightweight, [RosBridgeJS](https://github.com/klpanagi/RosBridgeJS) module.
69 This module has been developed under the RAPP Project to provide to HOP Services developers, an **easy-to-use** intermediate transport interface layer for calling ROS-Services through their hopjs applications.
70 It integrates a websocket client that allows connections to [rosbridge-websocket-server](http://wiki.ros.org/rosbridge_server) and a service controller to call ROS-Service(s).
71 
72 Initiate rosbridgejs:
73 
74 ```javascript
75 /***
76  * Import the RosBridgeJS module.
77  */
78 var Rosbridge = require( '<path_to_RosBridgeJs_sources>/RosBridgeJS.js' );
79 
80 /***
81  * Initiate connection to rosbridge_websocket_server.
82  *
83  * By default, it tries to connect to ws://localhost:9090.
84  * This is the default URL the rosbridge-websocket-server listens to.
85  */
86 var ros = new ROS({hostname: '', port: '', reconnect: true, onconnection:
87  function(){
88  // .
89  }
90 });
91 
92 ```
93 
94 
95 Fill request parameters for the ROS-Service:
96 
97 ```javascript
98  /***
99  * Fill Ros Service request msg parameters here.
100  * var args = {param1: '', param2: ''};
101  *
102  */
103  var args = {
104  imageFilename: 'face_image_frame.png'
105  };
106 ```
107 
108 Declare/Implement the callback functions:
109 
110 ```javascript
111 
112  /***
113  * Declare the ROS-Service response callback here!!
114  * This callback function will be passed into the rosbridge service
115  * controller and will be called when a response from rosbridge
116  * websocket server arrives.
117  */
118  function callback(data){
119  console.log(data);
120  }
121 
122  /***
123  * Declare the onerror callback.
124  * The onerror callack function will be called by the service
125  * controller as soon as an error occures, on service request.
126  *
127  */
128  function onerror(e){
129  console.log(e);
130  }
131 
132 ```
133 
134 Call the ROS-Service (/rapp/rapp_face_detection/detect_faces):
135 
136 ```javascript
137 
138  var rosSrvName = '/rapp/rapp_face_detection/detect_faces';
139 
140  /***
141  * Call ROS-Service.
142  * Input arguments:
143  * - rosSrvName: The name of the ROS-Service to call</li>
144  * - args: ROS-Service request message arguments</li>
145  * - objLiteral: An object literal to declare the onsuccess and
146  * onerror callbacks.
147  * { success: <onsuccess_callback>,fail: <onerror_callback> }
148  *
149  */
150  ros.callService(rosSrvName, args,
151  {success: callback, fail: onerror});
152 ```
