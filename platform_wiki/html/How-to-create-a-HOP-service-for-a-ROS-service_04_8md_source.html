1 RAPP Web Services are implemented on top of the [hop.js](http://hop.inria.fr/home/index.html) framework.
2 > Hop.js is a multitier extension of JavaScript. It allows a single JavaScript program to describe the client-side and the server-side components of a Web application.
3  Its runtime environment ensures a consistent execution of the application on the server and on the client.
4  Hop programs execute in the context of a builtin web server. They define services, which are super JavaScript functions that get automatically invoked when HTTP requests are received.
5 
6 A framework has been developed, build ontop of hop.js, for easily implementing Web Services for the RAPP Platform. Zero knowledge of hop.js is required.
7 Additionally, Web Services are fully parametrized through single configuration files:
8 
9 - [services.json](https://github.com/rapp-project/rapp-platform/blob/master/rapp_web_services/config/services/services.json)
10 - [workers.json](https://github.com/rapp-project/rapp-platform/blob/master/rapp_web_services/config/services/workers.json)
11 
12 If you are not familiar with server-side applications, you might also have to read on [Nodejs](https://nodejs.org/en/).

15 ## RAPP Web Services framework
16 
17 For easily implementing and launching Web Services for the RAPP Platform, the RAPP Web Services framework has been developed. It consists of a [WebService](https://github.com/rapp-project/rapp-platform/tree/master/rapp_web_services/src/webService) implementation. build ontop of hop.js and an engine that allows to assign specific Web Services to Worker threads (Web Workers).
18 
19 
20 ### Web Service implementation
21 
22 Web Services are implemented in a single function implementation:
23 
24 ```js
25 function svcImpl(req, resp, ros){
26  // Web service implementation here.
27 }
28 ```
29 
30 This is the onrequest callback function to feed to the engine and will be called as soon as a request arrives.
31 
32 > The **req** (request) and **res** (response) objects are passed so you can access the request properties and craft and return responses. Additionally a **ros** object is passed that allows connections to ROS thought the rosbridge-websocket transport layer.
33 
34 The `req` object has the following properties:
35 - `header`: Request header.
36 
37 ```js
38 console.log(req.header)
39 >> {
40 >> host: 'localhost:9001',
41 >> content-length: 679,
42 >> accept-encoding: 'gzip, deflate',
43 >> accept: '*/*',
44 >> user-agent: 'rapp-platform-api/python',
45 >> accept-token: 'rapp_token',
46 >> connection: 'keep-alive',
47 >> content-type: 'multipart/form-data; boundary=595d1046de7f4c958ca662c37140215a'
48 >> }
49 ```
50 
51 - `socket`: Connection socket
52 
53 ```js
54 console.log(req.socket)
55 >> {
56 >> hostname: 'localhost',
57 >> hostAddress: '127.0.0.1',
58 >> localAddress: '127.0.0.1',
59 >> port: 43661
60 >> }
61 ```
62 
63 - `body`: Request body
64 
65 ```js
66 console.log(req.body)
67 >> { fast: true }
68 ```
69 
70 - `files`: In case of uploading files, this field contains the paths to the uploaded files. Access the files by name using dot notation.
71 For example a service receives a single-file in fieldname `single_file` and an array of files in fieldname `file_array`:
72 
73 ```js
74 console.log(req.files)
75 >> {
76 >> single_file: ["PATH"],
77 >> file_array: ["PATH_FILE_1", "PATH_FILE_2"]
78 >> }
79 ```
80 
81 **Note**: Note that even if it is a **single-file**, the `single_file` property of `req.files` (`req.files.single_file`) is an Array.
82 
83 - `username`: This is the **username** of the client that requested access to the RAPP Platform resources (Services). It is automatically applied to the `req` object, after appliance of the [RAPP Authentication](https://github.com/rapp-project/rapp-platform/wiki/RAPP-Application-Authentication) on request arrival. Note that before execution of the onrequest callback function, we apply authentication to the request. If the authentication is not successful, an **HTTP 401 Unauthorized** error is returned to the client.
84 
85 ```js
86 console.log(req.username)
87 >> "RAPP_USER"
88 ```
89 
90 The `resp` object has the following properties (methods):
91 
92 - sendJson(obj): Send an application/json response
93 
94 ```js
95 function svcImpl(req, resp, ros){
96  ...
97 
98  var response = {error: ''};
99  resp.sendJson(response);
100 }
101 ```
102 
103 - `sendServerError()`: Respond with **HTTP 500 Internal Server Error**
104 
105 ```js
106 function svcImpl(req, resp, ros){
107  ...
108 
109  resp.sendServerError();
110 }
111 ```
112 
113 - `sendUnauthorized()`: Respond with **HTTP 401 Unauthorized Client Error**
114 
115 ```js
116 function svcImpl(req, resp, ros){
117  ...
118 
119  resp.sendUnauthorized();
120 }
121 ```

125 ### Web Service configuration and registration.
126 
127 Web Services are fully parametrized through the [services.json](https://github.com/rapp-project/rapp-platform/blob/master/rapp_web_services/config/services/services.json) file. This file includes Web Services to be launched (along with the Web Service parameters),
128 that was previously declared in the [workers.json](https://github.com/rapp-project/rapp-platform/blob/master/rapp_web_services/config/services/workers.json) file.
129 
130 Below is the face_detection entry:
131 
132 ```js
133 "face_detection": {
134  "launch": true,
135  "anonymous": false,
136  "name": "face_detection",
137  "url_name": "face_detection",
138  "namespace": "",
139  "ros_connection": true,
140  "timeout": 45000
141 }
142 ```
143 
144 **Web Service configuration parameters**:
145 
146 - `launch` (Boolean): If true this Web Service will be launched.
147 - `anonymous` (Boolean): If true, this service will be anonymous, which means that it will be assigned a random url path.
148 - `name` (String): The service name.
149 - `urlname` (String): The service urlname. Service name can be different from the urlname.
150 - `namespace` (String): Namespace for the urlname to append as a prefix to the service url name. For example, a service with **urlname="faca_detection"** and **namespace="computervision"** will be translated to **/computervision/face_detection**
151 - `timeout` (String): Request timeout value.
152 - `ros_connection` (Boolean): If true, a ros object that allowes for calls to the ROS Services will be passed to the onrequest callback function.

155 ### Run a Web Service within an existing Web Worker
156 
157 Web services run within server-side workers (Web Workers). A worker can include more than one web service. We consider server-side workers to be forked processes, thus allowing concurrent execution.
158 
159 To run a Web Service within a Web Worker, just specify the service name in the **services** field of the worker in the [worker.json](https://github.com/rapp-project/rapp-platform/blob/master/rapp_web_services/config/services/workers.json) file.
160 
161 For example, the **weather_report** worker holds the **weather_report_current** and **weather_report_forecast** Web Services:
162 
163 ```js
164 "weather_report": {
165  "launch": true,
166  "path": "workers/weather_report.js",
167  "services": [
168  "weather_report_forecast",
169  "weather_report_current"
170  ]
171 }
172 ```
173 
174 **Web Worker configuration parameters**:
175 
176 - `launch` (Boolean): Weather to launch the Web Worker or not.
177 - `path` (String): Path to the Web Worker source file. Relative to the **rapp_web_services** directory
178 - `services`: (Array): Services to launch under the Web Worker thread.

181 ### Where to store Web Service implementation source file(s) and how to launch it.
182 
183 Source files are stored under the [services](https://github.com/rapp-project/rapp-platform/tree/master/rapp_web_services/services) directory, of the [rapp_web_services](https://github.com/rapp-project/rapp-platform/tree/master/rapp_web_services) package.
184 
185 Web Services are automatically loaded from single .js files, as node.js modules. Make sure you export the Web Service implementation function:
186 
187 ```js
188 function svcImpl(req, resp, ros){
189  // Web service implementation here.
190 }
191 
192 ...
193 
194 module.exports = svcImpl;
195 
196 ```

198 ### Complete Web Service Implementation Example - Face Detection
199 
200 The following example illustrates the implementation of a WebService that connects to the Face-Detection RAPP-Platform backend Service
201 
202 **ROS Service Message**:
203 ```bash
204 # Contains info about time and reference
205 Header header
206 # The image's filename to perform face detection
207 string imageFilepath
208 # Flag to define if a fast detection if desired
209 bool fast
210 ---
211 # Container for detected face positions
212 geometry_msgs/PointStamped[] faces_up_left
213 geometry_msgs/PointStamped[] faces_down_right
214 string error
215 ```
216 
217 and the Face-Detection ROS Service url path is: `/rapp/rapp_face_detection/detect_faces`
218 
219 **Web Service Request**:
220 
221 - `file`: Image file.
222 - `fast` (Bool): If true, detection will take less time but it will be less accurate.
223 
224 **Web Service Response**:
225 
226 - `faces` (Array): Detected faces.
227 - `error` (String): Error message.
228 
229 
230 First, create the face_detection `svc.js` file:
231 
232 ```bash
233 $ cd ~/rapp_platform/rapp-platform-catkin-ws/src/rapp-platform/rapp_web_services/services
234 $ mkdir face_detection && cd face_detection
235 $ touch svc.js
236 ```
237 
238 Open the svc.js file with your favorite editor:
239 
240 ```bash
241 $ vim svc.js
242 ```
243 
244 Implement the structure of the `client-response` and `ros-msg` objects:
245 
246 ```js
247 var clientRes = function(faces, error) {
248  return { faces: [], error: '' }
249 }
250 
251 var rosReqMsg = function(imageFilepath, fast) {
252  return { imageFilepath: '', fast: false }
253 }
254 ```
255 
256 Assign ROS Service url path to a global variable:
257 
258 ```js
259 var rosSrvUrlPath = "/rapp/rapp_face_detection/detect_faces";
260 ```
261 
262 Next, implement the service onrequest callback function:
263 
264 ```js
265 function svcImpl(req, resp, ros) {
266  // If no image file received, return to client with an error
267  if( ! req.files.file ){
268  // Create a client response object
269  response = new client_res();
270  response.error = "No image file received";
271  // Send response (application/json)
272  resp.sendJson(response);
273  return;
274  }
275 
276  // Create a ROS Service Request Message and fill values from client request
277  var rosMsg = new rosReqMsg();
278  rosMsg.imageFilename = req.files.file[0];
279  rosMsg.fast = req.body.fast;
280 
281  /***
282  * ROS-Service response callback.
283  */
284  function callback(data){
285  // Delete image file from the Platform cache directory.
286  fs.exists(_filepath, function(exists){
287  if(exists){
288  fs.unlink(_filepath)
289  }
290  })
291  // Parse rosbridge message and craft client response
292  var response = parseRosbridgeMsg( data );
293  resp.sendJson(response);
294  }
295 
296  /***
297  * ROS-Service onerror callback.
298  */
299  function onerror(e){
300  // Delete image file from the Platform cache directory.
301  fs.exists(_filepath, function(exists){
302  if(exists){
303  fs.unlink(_filepath)
304  }
305  })
306  // Respond a "Server Error". HTTP Error 501 - Internal Server Error
307  resp.sendServerError();
308  }
309 
310  /***
311  * Call ROS-Service.
312  */
313  ros.callService(rosSrvUrlPath, rosMsg,
314  {success: callback, fail: onerror});
315 }
316 
317 function parseRosbridgeMsg( rosbridge_msg )
318 {
319  var faces_up_left = rosbridge_msg.faces_up_left;
320  var faces_down_right = rosbridge_msg.faces_down_right;
321  var error = rosbridge_msg.error;
322  var numFaces = faces_up_left.length;
323 
324  // Create a new response object
325  var response = new client_res();
326 
327  if( error ){
328  // If ROS Service responded with an error
329  response.error = error;
330  return response;
331  }
332 
333  for (var ii = 0; ii < numFaces; ii++)
334  {
335  var face = {
336  up_left_point: {x: 0, y:0},
337  down_right_point: {x: 0, y: 0}
338  };
339 
340  face.up_left_point.x = faces_up_left[ii].point.x;
341  face.up_left_point.y = faces_up_left[ii].point.y;
342  face.down_right_point.x = faces_down_right[ii].point.x;
343  face.down_right_point.y = faces_down_right[ii].point.y;
344  response.faces.push( face );
345  }
346 
347  return response;
348 }
349 ```
350 
351 Export the service onrequest callback function (`svcImpl`):
352 
353 ```js
354 module.exports = svcImpl
355 ```
356 
357 Next you will need to create the Web Worker to launch the Web Service:
358 
359 ```bash
360 $ cd ~/rapp_platform/rapp-platform-catkin-ws/src/rapp-platform/rapp_web_services/workers
361 $ touch face_detection.js
362 ```
363 
364 Open the **face_detection.js** file with your favorite editor:
365 
366 ```bash
367 $ vim face_detection.js
368 ```
369 
370 Import the workerUtils module:
371 
372 ```js
373 var path = require('path');
374 
375 var ENV = require( path.join(__dirname, '../..', 'env.js') );
376 
377 // Include it even if not used!!! Sets properties to the thread's global scope.
378 var workerUtils = require(path.join(ENV.PATHS.INCLUDE_DIR, 'common', 'worker_utils.js'));
379 ```
380 
381 Next, you will have to set the worker name and call to launch all services registred to this Web Worker:
382 
383 ```js
384 // Set worker thread name under the global scope. (WORKER.name)
385 workerUtils.setWorkerName('face_detection');
386 
387 // Launch all services assigned to this worker thread.
388 // Search in workers.json config file for assigned web services.
389 workerUtils.launchSvcAll();
390 ```
391 
392 We need to tell the *run-engine* to launch the, newly implemented, Web Worker.
393 
394 The `workers.json` file containes Web Workers entries. It is located under:
395 
396 ```bash
397 ~/rapp_platform/rapp-platform-catkin-ws/src/rapp-platform/rapp_web_services/config/services
398 ```
399 
400 Append the following entry in the workers.json file:
401 
402 ```js
403 "face_detection": {
404  "launch": true,
405  "path": "workers/face_detection.js",
406  "services": [
407  "face_detection"
408  ]
409 }
410 ```
411 
412 Finally append the following *web-service* entry in the services.json file (under the same directory):
413 
414 ```js
415 "face_detection": {
416  "launch": true,
417  "anonymous": false,
418  "name": "face_detection",
419  "url_name": "face_detection",
420  "namespace": "",
421  "authentication": true,
422  "ros_connection": true,
423  "timeout": 45000
424 }
425 ```
426

431 $ pm2 start server.yaml
432 ```
