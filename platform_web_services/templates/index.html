RAPP Platform Web Services Development Templates

RAPP Platform Web Services Development Templates

Synopsis

Use these templates/examples as the ground-truth to develop and/or extend RAPP Platform Web Services (a.k.a HOP Web Services).

While developing RAPP Platform Web Services, you might want to read on HOP/hopjs

If you are not familiar with server-side applications, you might also have to read on Nodejs

Templates

To make development of HOP Web Services more attractive and understandable, we provide two template source files:

  • web_service.template.js
  • web_service_post_file.template.js

Using these templates as a reference while developing HOP Web Services, might save you alot of hours studying!
In most cases, you might have to change just a few lines of code!

web_service.template.js

Illustrates a web-service implementation with, multiple input arguments, ROS interface, asynchronous response.

web_service_post_file.template.js

Illustrates a web-service implementation with, multiple input arguments, file transfers, ROS interface, asynchronous response.

web_service_post_file.template.js

Illustrates the implementation of a HOP Service, a.k.a
RAPP-Platform-Web-Service, with file post field(s).

We use multipart/form-data HTTP POST requests to transfer files from
client-side applications to the RAPP Platform Web Services.

An example of using multipart/form-data file-data post fields is a request
for RAPP Platform face-detection service. In this case we need to send an
image frame to the face_detection RAPP Platform service in order to perform
face_detection operations onto that image frame.

While posting multipart/form-data requests, the Front-End HOP HTTP Server
is responsible to parse the post fields, export the file-data post field,
store the file into the server's cache directory and report, as input
to the relevant HOP Web Service, the system path to the stored file.
